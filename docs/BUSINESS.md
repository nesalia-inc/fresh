# Business Strategy

## The Vision

Fresh doesn't sell documentation. It sells **"optimized agent brain time"**.

In a world where AI can generate millions of pages of text, **verified, short, structured data** becomes the new gold.

## The Core Insight

> You don't sell doc. You sell the time Fresh saves your agents.

Every token Fresh saves an enterprise = direct money in their pocket.

---

## Revenue Streams

### 1. Registry Premium (B2C & B2B)

**Model: GitHub/NPM for knowledge**

#### B2C: Individuals
- Free: Public guides
- Pro: Unlimited private guides (your "second brain" stays secret)

#### B2B: Private Registry
Companies have terrible internal docs (Notion, Slack, Confluence).

**Offer:** Secure private registry for:
- Company standards
- Architecture docs
- Onboarding guides

**Value:** Claude Code can `fresh registry pull @company/internal-api` and become instantly expert on proprietary code. Massive productivity gain.

### 2. "AI-Ready" Certification (B2B)

Libraries want AI to code well with them (Vercel, Meta, etc.)

**Offer:** Paid certification service
- Help DevRel teams transform messy HTML docs into Fresh guides optimized for LLMs
- Structured, with tested code examples

**Revenue:** Companies pay to be "Featured" or "Official" in Registry.

### 3. Context-as-a-Service

If Fresh becomes the reference for structured doc:

**API for AI tools:**
- IDE plugins (Cursor, etc.) pay for API access to improve their models

**Fresh Cloud Sync:**
- Pay for automatic sync between machines
- Start Learning Project at office → exact same state on laptop

### 4. Learning Paths (Result-Based Selling)

Recruiting and training developers costs €€€.

**Offer:** Certified "Skill Packs"
- Example: "Senior Go Developer Pack" = 50 structured Fresh guides
- Target: Training platforms, hiring companies

**Revenue:** License per developer.

## Go-to-Market

### 1. Open Source (CLI)
Build adoption and user base.

### 2. SaaS (Registry)
- Free: Public guides
- Paid: Private guides (Teams/Enterprises)

### 3. MCP Connector (Growth Lever)
Build "Fresh MCP Server" for Claude Desktop and Code to use Fresh natively.

## Why It Works

| Traditional Doc | Fresh |
|-----------------|-------|
| Millions of pages | Curated, verified guides |
| AI wastes time searching | AI gets structured answers |
| Token expensive | Token saved = money saved |

**Fresh = "Time saved for AI agents" = Direct enterprise value**

## Competitive Advantage

Others can copy the CLI. But:
- Community guides (network effect)
- Official certifications (relationships with DevRel)
- Curated learning paths (content strategy)

It's not the tool. It's the **curation**.
